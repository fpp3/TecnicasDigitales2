\chapter{Desarrollo}

El proyecto principal consistió en implementar un \textbf{Blink LED} utilizando el LED 
integrado en la placa BluePill, el cual se encuentra conectado al pin \textbf{PC13}. A 
continuación se detallan los aspectos técnicos fundamentales del desarrollo.

\section{Distribución de Memoria y Linker Script}

El archivo \texttt{linker.ld} es responsable de indicar al enlazador cómo organizar las 
secciones del programa en la memoria física del microcontrolador. En el bloque 
\texttt{MEMORY} se definen las regiones disponibles:

\begin{itemize}
    \item \textbf{FLASH}: Memoria de solo lectura ($rx$) que comienza en $0x08000000$ con 
    un tamaño de $64 \kB$. Aquí se almacenan el vector de interrupciones y el código ejecutable.
    \item \textbf{RAM}: Memoria de lectura y escritura ($xrw$) que comienza en 
    $0x20000000$ con un tamaño de $20 \kB$.
\end{itemize}

Además, se define el símbolo \texttt{\_esstack} al final de la RAM para inicializar el 
puntero de pila (\textit{Stack Pointer}) durante el arranque:

\begin{lstlisting}[language=C]
_esstack = ORIGIN(RAM) + LENGTH(RAM);
\end{lstlisting}

\section{Uso de Registros Atómicos}

Para la manipulación de los pines de salida, se prefirió el uso del registro 
\texttt{BSRR} (\textit{Bit Set/Reset Register}) sobre el registro de salida 
\texttt{ODR}. El registro \texttt{BSRR} permite realizar operaciones atómicas de 
escritura:

\begin{enumerate}
    \item \textbf{Seguridad en hilos/interrupciones}: No requiere el ciclo de 
    lectura-modificación-escritura (LME), evitando condiciones de carrera si una 
    interrupción modifica otro pin del mismo puerto durante la operación.
    \item \textbf{Eficiencia}: Se puede poner un pin en alto escribiendo en los 16 bits 
    bajos del registro, o en bajo usando los 16 bits altos (offset de 16).
\end{enumerate}

\section{Depuración con OpenOCD y GDB}

El flujo de trabajo para la depuración \textit{bare-metal} se basa en una arquitectura 
cliente-servidor:

\begin{enumerate}
    \item \textbf{OpenOCD}: Actúa como servidor, comunicándose con el programador ST-Link 
    v2 mediante el protocolo SWD y exponiendo una interfaz GDB en el puerto local 3333.
    \item \textbf{GDB}: El cliente de depuración (\texttt{arm-none-eabi-gdb}) se conecta 
    al servidor para inspeccionar registros, memoria y controlar la ejecución.
\end{enumerate}
